# LVM Graph Viewer

![Banner](banner.png)

Независимое от внешних библиотек десктопное приложение на C++ для просмотра файлов сигналов LabVIEW `.lvm` / `.txt`.

**[🇷🇺 Русский](README_RU.md)** | **[🇬🇧 English](README_EN.md)**

## Обзор

Два фронтенда используют общую библиотеку парсера/БПФ:

- **`lvm_viewer_gui.exe`** — нативное десктопное приложение Win32 (окно, кнопки, интерактивный график). См. [GUI просмотрщик](#gui-просмотрщик).
- **`lvm_reader.exe`** — консольный инструмент (структура, статистика, пики БПФ, экспорт CSV). См. [Использование CLI](#использование-cli).

Парсер — точный порт на C++ Python-версии LVM Signal Viewer, а БПФ совпадает с `numpy.rfft` (погрешность ~1e-13). Результаты совпадают с Python-референсом на прилагаемых тестовых файлах (проверено на файле 1 ГБ / 6.8 млн отсчётов).

## GUI Просмотрщик

Автономное приложение, нарисованное через Win32 API + GDI (без Qt, без DLL). Последняя версия: **v0.4.4**.

### Приветственный экран

При запуске (без файла) показывается окно приветствия с названием приложения, краткой инструкцией и кнопками: открыть файл, настройки точек, справка по горячим клавишам или начать работу.

### Основной интерфейс

- **Главное меню** (Файл / Вид / Точки / Линии / Маркеры / Справка) вместе с панелью инструментов, современный дизайн на Segoe UI и строка статуса.
- **Тёмная тема** — переключение клавишей `T` или через меню Вид. Полная тёмная тема: панель инструментов, меню, боковая панель, окно настроек.
- **Язык** — переключение между русским и английским через меню Вид.
- **Открыть файл** — открыть `.lvm`/`.txt` через диалог (или передать путь в командной строке / перетащить файл на exe).
- Многоканальный график с цветной **интерактивной легендой** — клик по строке легенды включает/выключает канал, `Ctrl+Клик` — solo (показать только этот канал).
- **Панель каналов** — чекбоксы для показа/скрытия каждого канала.
- **Время / Гц** — переключение между графиком сигнала и спектром БПФ.
- **Сохранить PNG / Сохранить CSV** — экспорт текущего вида: PNG-изображение (GDI+) или CSV **видимого сегмента** (временное окно в режиме Время или видимая полоса спектра в режиме Гц). Имена по умолчанию отражают режим (`…_plot.png` / `…_spectrum.png`, `…_segment.csv` / `…_spectrum.csv`).
- **▶ Воспроизведение / ⏸ Пауза** — воспроизведение сигнала в **реальном времени** (1 с сигнала за 1 с реального времени); красная линия playhead движется по графику, и когда она проходит середину, окно плавно прокручивается вслед за ней.
- **Точки** — инструмент измерения. Кликайте по графику, чтобы ставить маркеры; значения отображаются прямо на графике и в строке статуса. Отдельная **панель «Настройки точек измерения»** (кнопка на панели или *Точки → Настройки…*) содержит чекбоксы: номер точки, X, Y, расстояние по X (Δx) и/или Y (Δy), 1/Δt, прямолинейное расстояние d — а также **примагничивание к графику** и кнопку **смены цвета маркеров**. Правый клик (или `Delete`) очищает точки.
- **Линии** — добавляйте **вертикальные** и **горизонтальные** направляющие линии на график. Выберите *Добавить вертикальную/горизонтальную линию*, затем кликните по нужному месту (вертикальные линии примагничиваются к отсчёту, если включено примагничивание); *Очистить линии* удаляет их. Линии запоминаются отдельно для каждого режима (Время / Гц).
- **Auto zoom** — по умолчанию масштаб по высоте подстраивается под видимые данные; эта кнопка фиксирует текущий диапазон, чтобы высота перестала перестраиваться (удобно при воспроизведении / прокрутке).
- **Визуальное сглаживание (сплайн)** — *чисто визуальная* кривая Кэтмулла-Рома, проведённая между отсчётами. Она сглаживает **линию**, не смещая реальные точки данных (измерения и примагничивание по-прежнему попадают в истинные отсчёты); при увеличении масштаба реальные отсчёты показываются точками, чтобы различие было очевидно.
- **Мелкая сетка** — тонкие линии сетки между основными делениями осей для удобства чтения.
- **Сбросить вид** + масштабирование колёсиком мыши (под курсором) и панорамирование ЛКМ. Масштаб/панорама работают по **обеим** осям — времени в режиме Время и **частоте в режиме Гц** — так что можно приблизить узкую полосу. Строка статуса показывает текущее окно и число отсчётов.

### Горячие клавиши

Полный список доступен в приложении через **Справка → Горячие клавиши** (или `F1`).

| Клавиша | Действие |
|---------|----------|
| `O` / `Ctrl+O` | Открыть файл |
| `S` / `Ctrl+S` | Сохранить PNG |
| `E` / `Ctrl+E` | Сохранить CSV |
| `M` | Переключить Время / Гц |
| `Пробел` | Воспроизведение / Пауза |
| `V` | Режим точек вкл/выкл |
| `C` | Сглаживание сплайном вкл/выкл |
| `+` / `↑` | Увеличить |
| `−` / `↓` | Уменьшить |
| `←` / `→` | Сдвиг влево / вправо |
| `Home` | Сбросить вид |
| `Delete` | Очистить точки измерения |
| `Esc` | Отменить добавление линии |
| `F1` | Справка по горячим клавишам |
| `T` | Переключить тёмную/светлую тему |
| `Ctrl+Z` | Отменить |
| `Ctrl+Shift+Z` | Повторить |

Мышь: колёсико масштабирует под курсором, тяга ЛКМ — панорамирование, левый клик — поставить точку или линию (в соответствующем режиме), правый клик — очистить точки. Клик по легенде — переключить канал, Ctrl+Клик — solo.

### Сборка (Windows)

```powershell
powershell -ExecutionPolicy Bypass -File build_gui.ps1
```

или через `make gui`, или напрямую:

```bash
g++ -std=c++17 -O2 -municode -static -mwindows -o lvm_viewer_gui.exe \
    gui_main.cpp lvm_parser.cpp fft.cpp analysis.cpp \
    -lcomdlg32 -lgdi32 -luser32 -lgdiplus -lcomctl32
```

Затем запустите `lvm_viewer_gui.exe` двойным кликом. (Закройте приложение перед пересборкой — запущенный exe не может быть перезаписан компоновщиком.)

## Использование CLI

Требуется компилятор C++17. Протестировано с MSYS2/MinGW `g++ 14.2`.

Через Makefile:

```bash
make          # собрать lvm_reader(.exe)
make test     # собрать и запустить юнит-тесты
```

Или напрямую g++ (статическая линковка делает бинарник автономным — не нужны DLL `libstdc++`/`libgcc` во время выполнения):

```bash
g++ -std=c++17 -O2 -static -o lvm_reader.exe \
    main.cpp lvm_parser.cpp fft.cpp analysis.cpp
```

> Примечание: на этом тулчейне MSYS2 динамическая линковка падает (`ld returned 116`); `-static` обходит проблему и создаёт портативный бинарник.

```text
lvm_reader <файл.lvm|файл.txt> [опции]

Действия:
  -i, --info          Показать структуру файла / заголовок
  -s, --stats         Показать статистику по каналам (мин/макс/среднее/количество)
  -H, --head N        Вывести первые N строк данных
  -c, --csv ФАЙЛ      Экспортировать данные в CSV
      --fft           Показать сильнейшие спектральные пики по каналам
      --peaks N       Число пиков (по умолчанию 5; включает --fft)
      --fft-csv ФАЙЛ  Экспортировать амплитудный спектр в CSV

Выборка (применяется к stats / head / csv / fft):
      --start T       Оставить строки с временем >= T
      --end T         Оставить строки с временем <= T
      --channels СПИСОК Список позиций каналов через запятую, 1-индекс, например 1,3

Параметры парсинга / спектра:
  -m, --monotonic     Перестроить монотонную временную шкалу (для секционированных файлов)
      --keep-dup-time Оставить каналы, дублирующие ось времени
      --fft-samples N Проредить до N отсчётов для БПФ (0 = использовать все)
  -v, --verbose       Подробный вывод парсера
  -h, --help          Показать эту справку
```

Без флага действия показываются `--info` и `--stats`.

### Быстрый запуск на Windows (двойной клик)

`run.bat` — удобный лаунчер: дважды кликните и введите/вставьте путь к файлу, или перетащите `.lvm`/`.txt` файл на него. Запускает `--info --stats --fft` и оставляет окно открытым до нажатия клавиши. (Батник вызывает exe по полному пути, так что работает даже на машинах, где запуск программ из текущей папки запрещён.)

### Примеры

```bash
# Структура + статистика (по умолчанию)
./lvm_reader.exe lvm_files_for_tests/test.lvm

# Первые 5 строк
./lvm_reader.exe lvm_files_for_tests/test.lvm --head 5

# Экспорт в CSV
./lvm_reader.exe lvm_files_for_tests/test.lvm --csv out.csv

# Спектральные пики по каналам
./lvm_reader.exe lvm_files_for_tests/test.lvm --fft --peaks 3

# Экспорт спектра только для первого канала
./lvm_reader.exe lvm_files_for_tests/test.lvm --channels 1 --fft-csv spec.csv

# Временное окно первого канала
./lvm_reader.exe lvm_files_for_tests/test.lvm --start 0 --end 0.5 --channels 1 --csv window.csv

# Многосекционный файл с перестроенной монотонной шкалой
./lvm_reader.exe lvm_files_for_tests/test1.lvm --info --monotonic
```

## Примечания по парсингу

Поведение совпадает с Python-версией:

- Строки, начинающиеся с `***`, и известные метаданные (например, `Separator`, `Multi_Headings`, `X0`) пропускаются.
- Десятичные запятые нормализуются в точки; столбцы разделяются табуляцией.
- Строка должна содержать минимум два числовых значения, чтобы считаться данными; пропущенные/некорректные ячейки становятся `NaN`, так что выравнивание каналов сохраняется.
- Первый столбец считается временем/X; остальные становятся `Channel_1`, `Channel_2`, … Каналы без реальных значений удаляются.
- Строки с нечисловым значением времени удаляются.
- Столбцы, просто дублирующие ось времени, удаляются (отключить через `--keep-dup-time`).
- `--monotonic` выравнивает секции `Multi_Headings`, которые сбрасывают локальное время.

## Примечания по спектру

- Частота дискретизации выводится из медианы положительных шагов времени, сигнал обнуляется по среднему, амплитуда — `2/N * |rfft|` — как в Python-режиме Гц.
- По умолчанию преобразуется весь выбор (Bluestein FFT работает с любым N). Для очень больших файлов `--fft-samples N` равномерно прореживает данные для ускорения; прореживание сохраняет корректность частотной оси (падает предел Найквиста).

## Файлы

- `lvm_parser.hpp` / `lvm_parser.cpp` — библиотека парсера
- `fft.hpp` / `fft.cpp` — примитивы БПФ радикса-2 + Блюстейна
- `analysis.hpp` / `analysis.cpp` — спектр + поиск пиков
- `gui_main.cpp` — нативный GUI-просмотрщик Win32
- `build_gui.ps1` — скрипт сборки GUI (Windows/PowerShell)
- `main.cpp` — консольный фронтенд
- `run.bat` — лаунчер для CLI: двойной клик / drag-and-drop (Windows)
- `tests/run_tests.cpp` — юнит-тесты (`make test`)
- `Makefile` — вспомогательный файл сборки
